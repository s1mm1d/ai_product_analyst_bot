# ai_product_analyst_bot
Бот для дизайна продуктовых исследований по запросу

# тег бота @ai_research_project_bot

## Описание проекта

Этот n8n workflow представляет собой Telegram-бота, который помогает создавать структурированные планы продуктовой аналитики на основе описания задачи от пользователя.

---

## Как работает бот

### Логика взаимодействия с пользователем

#### 1. Первое сообщение пользователя

* Если пользователь отправляет что-то кроме `/form_problem` → бот просит начать с команды `/form_problem`
* Если пользователь отправляет `/form_problem` → бот:

  * отправляет описание своих возможностей
  * просит описать аналитическую задачу

#### 2. Второе сообщение (описание задачи)

* Бот проверяет, что пользователь уже использовал команду `/form_problem`
* Обрабатывает описание задачи через полный аналитический pipeline
* Возвращает структурированный план анализа

#### Обработка ошибок

* Если описание задачи некорректное или недостаточно детальное → бот останавливает обработку и просит уточнить

---

## Что делает бот

Бот анализирует описание аналитической задачи и создает детальный план, включающий:

* Формулировку проблемы (Problem Statement)
* Гипотезы для проверки
* Метрики для измерения
* Требования к данным (источники, поля, фильтры)
* Сегменты для анализа
* План анализа (шаги выполнения)
* Рекомендации по визуализации
* Критический обзор плана с рисками и улучшениями

## Установка и настройка

### Шаг 1: Импорт workflow

1. Откройте n8n
2. Перейдите в раздел Workflows
3. Нажмите "Import from File"
4. Выберите полученный JSON файл

Workflow будет импортирован со всеми узлами и настройками.

---

### Шаг 2: Настройка Telegram

1. Создайте бота через @BotFather в Telegram
2. Получите API токен бота
3. В workflow откройте узел "Telegram Trigger"
4. Создайте новые credentials для Telegram или выберите существующие
5. Введите полученный API токен

---

### Шаг 3: Настройка Mistral AI

1. Получите API ключ от Mistral AI

2. В workflow найдите узлы с Mistral AI (их 7 штук):

   * Mistral Model - Frame
   * Mistral Model - Hypotheses
   * Mistral Model - Metrics
   * Mistral Model - Data
   * Mistral Model - Segments
   * Mistral Model - Plan
   * Mistral Model - Viz
   * Mistral Model - Review

3. В каждом узле:

   * создайте или выберите credentials для Mistral AI
   * введите ваш API ключ

---

### Шаг 4: Создание Data Tables

Важно: таблицы данных должны быть созданы вручную перед запуском workflow.

#### Таблица 1: Отслеживание состояния пользователей

Название: `user_states`

Колонки:

* `chat_id` (тип: text) — ID чата пользователя
* `last_command` (тип: text) — последняя команда пользователя
* `timestamp` (тип: number) — временная метка

Используется в узлах:

* Check User State
* Record Command Usage
* Update User State After Request

---

#### Таблица 2: Логирование ответов

Название: например, `analytics_responses`

Колонки:

* `formatted_response` (тип: text) — ответ бота
* `chat_id` (тип: text) — ID чата
* `user_id` (тип: text) — ID пользователя
* `timestamp` (тип: text) — временная метка
* `review_data` (тип: text) — данные критического обзора

Используется в узле:

* Log to Data Table

---

### Привязка таблиц к узлам

После создания таблиц:

* В узле "Check User State" выберите таблицу `user_states`
* В узле "Record Command Usage" выберите таблицу `user_states`
* В узле "Update User State After Request" выберите таблицу `user_states`
* В узле "Log to Data Table" выберите таблицу для логирования

---

### Шаг 5: Активация workflow

1. Убедитесь, что все credentials настроены
2. Убедитесь, что обе Data Tables созданы и привязаны
3. Нажмите кнопку "Active" в правом верхнем углу

После этого workflow начнет слушать сообщения из Telegram.

## Структура workflow

### Основные компоненты

* Telegram Trigger — получает сообщения
* Extract Message Data — извлекает данные
* Check Message Type — проверяет команду
* Check User State — проверяет состояние
* Has Used Command Before? — проверяет использование команды

---

### Аналитический pipeline

Состоит из 8 этапов (Mistral AI):

* Frame Problem
* Generate Hypotheses
* Select Metrics
* Identify Data Requirements
* Define Segments
* Create Analysis Plan
* Suggest Visualizations
* Review Plan

---

### Обработка результатов

* Format Response — форматирование
* Split Long Message — разбивка (лимит 4096 символов)
* Send to Telegram — отправка
* Log to Data Table — логирование

---

## Возможные проблемы и решения

### Бот не отвечает

* Проверьте, что workflow активирован
* Проверьте credentials Telegram
* Проверьте Data Tables

---

### Ошибка "Cannot read properties of undefined"

* Проверьте структуру таблиц
* Проверьте привязку таблиц

---

### Ошибка Mistral AI

* Проверьте API ключ
* Проверьте лимиты
* Убедитесь, что доступна модель `mistral-large-latest`

---

### Бот не запоминает состояние

* Проверьте таблицу `user_states`
* Проверьте узлы обновления состояния
