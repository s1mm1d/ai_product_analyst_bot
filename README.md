# ai_product_analyst_bot
Бот для дизайна продуктовых исследований по запросу

# тег бота @ai_research_project_bot

Описание проекта
Этот n8n workflow представляет собой Telegram-бота, который помогает создавать структурированные планы продуктовой аналитики на основе описания задачи от пользователя.

Как работает бот
Логика взаимодействия с пользователем
Первое сообщение пользователя:

Если пользователь отправляет что-то кроме /form_problem → бот просит начать с команды /form_problem
Если пользователь отправляет /form_problem → бот отправляет описание своих возможностей и просит описать аналитическую задачу
Второе сообщение (описание задачи):

Бот проверяет, что пользователь уже использовал команду /form_problem
Обрабатывает описание задачи через полный аналитический pipeline
Возвращает структурированный план анализа
Обработка ошибок:

Если описание задачи некорректное или недостаточно детальное, бот останавливает обработку и просит уточнить

Что делает бот
Бот анализирует описание аналитической задачи и создает детальный план, включающий:

Формулировку проблемы (Problem Statement)
Гипотезы для проверки
Метрики для измерения
Требования к данным (источники, поля, фильтры)
Сегменты для анализа
План анализа (шаги выполнения)
Рекомендации по визуализации
Критический обзор плана с рисками и улучшениями

Установка и настройка

Шаг 1: Импорт workflow
Откройте n8n
Перейдите в раздел Workflows
Нажмите "Import from File"
Выберите полученный JSON файл
Workflow будет импортирован со всеми узлами и настройками

Шаг 2: Настройка Telegram
Создайте бота через @BotFather в Telegram
Получите API токен бота
В workflow откройте узел "Telegram Trigger"
Создайте новые credentials для Telegram или выберите существующие
Введите полученный API токен

Шаг 3: Настройка Mistral AI
Получите API ключ от Mistral AI
В workflow найдите узлы с Mistral AI (их 7 штук):
Mistral Model - Frame
Mistral Model - Hypotheses
Mistral Model - Metrics
Mistral Model - Data
Mistral Model - Segments
Mistral Model - Plan
Mistral Model - Viz
Mistral Model - Review
В каждом узле создайте или выберите credentials для Mistral AI
Введите ваш API ключ

Шаг 4: Создание Data Tables
ВАЖНО: Таблицы данных должны быть созданы вручную перед запуском workflow.

Перейдите на вкладку Data Tables и создайте две таблицы:

Таблица 1: Отслеживание состояния пользователей
Название: user_states (или любое другое)

Колонки:

chat_id (тип: text) - ID чата пользователя
last_command (тип: text) - Последняя команда пользователя
timestamp (тип: number) - Временная метка
Эта таблица используется в узлах:

Check User State
Record Command Usage
Update User State After Request

Таблица 2: Логирование ответов
Название: выберите сами (например, analytics_responses)

Колонки:

formatted_response (тип: text) - Отформатированный ответ бота
chat_id (тип: text) - ID чата
user_id (тип: text) - ID пользователя
timestamp (тип: text) - Временная метка
review_data (тип: text) - Данные критического обзора
Эта таблица используется в узле:

Log to Data Table
Привязка таблиц к узлам
После создания таблиц:

Откройте узел "Check User State"

В параметре "Data Table" выберите таблицу user_states
Откройте узел "Record Command Usage"

В параметре "Data Table" выберите таблицу user_states
Откройте узел "Update User State After Request"

В параметре "Data Table" выберите таблицу user_states
Откройте узел "Log to Data Table"

В параметре "Data Table" выберите таблицу для логирования ответов

Шаг 5: Активация workflow
Убедитесь, что все credentials настроены
Убедитесь, что обе Data Tables созданы и привязаны
Нажмите кнопку "Active" в правом верхнем углу workflow
Workflow начнет слушать сообщения из Telegram

Структура workflow
Основные компоненты

Telegram Trigger - получает сообщения от пользователей

Extract Message Data - извлекает данные из сообщения

Check Message Type - проверяет, является ли сообщение командой /form_problem

Check User State - проверяет состояние пользователя в базе данных

Has Used Command Before? - определяет, использовал ли пользователь команду ранее

Аналитический pipeline
Состоит из 8 этапов, каждый использует Mistral AI:

Frame Problem - формулирует проблему
Generate Hypotheses - генерирует гипотезы
Select Metrics - выбирает метрики
Identify Data Requirements - определяет требования к данным
Define Segments - определяет сегменты
Create Analysis Plan - создает план анализа
Suggest Visualizations - предлагает визуализации
Review Plan - критически оценивает план
Обработка результатов
Format Response - форматирует ответ для пользователя
Split Long Message - разбивает длинные сообщения (лимит Telegram 4096 символов)
Send to Telegram - отправляет ответ пользователю
Log to Data Table - сохраняет результат в базу данных

Возможные проблемы и решения

Бот не отвечает
Проверьте, что workflow активирован (кнопка "Active")
Проверьте credentials для Telegram
Проверьте, что Data Tables созданы и привязаны

Ошибка "Cannot read properties of undefined"
Убедитесь, что все Data Tables созданы с правильными колонками
Проверьте, что таблицы привязаны к соответствующим узлам

Mistral AI возвращает ошибку
Проверьте, что API ключ Mistral AI действителен
Проверьте лимиты вашего аккаунта Mistral AI
Убедитесь, что модель mistral-large-latest доступна в вашем регионе

Бот не запоминает состояние пользователя
Проверьте, что таблица user_states создана
Проверьте, что узлы "Record Command Usage" и "Update User State After Request" привязаны к этой таблице
